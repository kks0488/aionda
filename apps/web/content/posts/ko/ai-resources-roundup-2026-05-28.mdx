## 세 줄 요약
- 네이버가 D2를 통해 Engineering Day 소식을 알리며 개발자 커뮤니티 접점을 이어갑니다.
- arXiv에서는 LLM 스케줄링, 편향 완화, 추론 데이터 압축 효과를 다루는 연구 3편이 나왔습니다.
- AWS와 주요 매체 기사들은 금융 문서 처리, 비즈니스 지원 에이전트, 데이터센터·콘텐츠 라벨링·멀티모달 생성 흐름을 함께 보여줍니다.

이번 글은 최근 24h 동안 수집된 자료를 기반으로 한 **링크 아카이브**다. 본문은 “요약 기사”가 아니라, 빠르게 원문으로 들어가기 위한 정리본이다.

## Official (공식)
- 🛡️ [모두의 NAVER Engineering Day! D2에서 이어집니다.](https://d2.naver.com/news/0766161) — NAVER D2
  - 왜 읽어야 하는가: 네이버 D2가 Engineering Day를 어떻게 이어가는지 제목만으로도 개발자 행사와 커뮤니티 방향을 빠르게 파악할 가치가 있습니다.
- 🏛️ [DynaSchedBench: Calibrated Dynamic Scheduling Benchmarks and Observability Paradox in LLM-based Scheduling Agents](https://arxiv.org/abs/2605.27566) — arXiv CS.AI
  - 왜 읽어야 하는가: LLM 기반 스케줄링 에이전트의 동적 벤치마크와 관측성 역설을 전면에 둔 제목이라 평가 설계 관점에서 읽어볼 만합니다.
- 🏛️ [Reward Bias Substitution: Single-Axis Bias Mitigations Redirect Optimization Pressure](https://arxiv.org/abs/2605.27996) — arXiv CS.AI
  - 왜 읽어야 하는가: 단일 축 편향 완화가 다른 최적화 압력을 어디로 옮기는지 다루는 제목이라 안전성 조정의 부작용을 점검하는 데 유용해 보입니다.
- 🏛️ [Zipping the Thought: When and How Compressed Reasoning Data Works in LLM Post-Training](https://arxiv.org/abs/2605.28008) — arXiv CS.AI
  - 왜 읽어야 하는가: 압축된 추론 데이터가 언제 어떻게 후학습에 효과적인지 겨냥한 제목이라 데이터 설계 효율을 고민하는 팀에 참고가 됩니다.
- 🏛️ [Process financial documents using Amazon Bedrock Data Automation](https://aws.amazon.com/blogs/machine-learning/process-financial-documents-using-amazon-bedrock-data-automation/) — AWS Machine Learning Blog
  - 왜 읽어야 하는가: Amazon Bedrock Data Automation으로 금융 문서를 처리하는 실무 맥락이 제목에 드러나 있어 문서 AI 적용 아이디어를 얻기 좋습니다.
- 🏛️ [Building AI agents for business support using Amazon Bedrock AgentCore](https://aws.amazon.com/blogs/machine-learning/building-ai-agents-for-business-support-using-amazon-bedrock-agentcore/) — AWS Machine Learning Blog
  - 왜 읽어야 하는가: 비즈니스 지원용 AI 에이전트를 Amazon Bedrock AgentCore로 구축하는 사례라 기업 업무 자동화 검토에 직접 연결됩니다.

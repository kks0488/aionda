A user writes a draft inside a search screen instead of copying one answer into a document.  
They revise the draft without leaving the search interface.  
In some cases, they assemble small tools alongside the draft.  
A canvas opens on one side of the screen.  
Plans, documents, and code can accumulate there.  
This suggests a shift in emphasis.  
Search becomes a place where work progresses.  
It becomes less focused on a single correct answer.  
Google describes this as.
“when you use Canvas in Search’s AI Mode, you can draft documents and even build interactive tools within search.”
This statement comes from an excerpt.

## TL;DR
- This describes a shift toward a canvas workspace inside Search AI Mode, including drafts and interactive tools.  
- It matters because saved drafts and possible code execution change trust, verification, and permissions needs.  
- Try small tasks in Canvas, and check history, settings, and any execution or network controls first.  

Example: You keep a working draft in search while gathering sources. You add a small helper tool for quick checks. You return later to revise and share.

## TL;DR
- The key issue for search AI is a UX shift.  
- It moves from an answer-centric UX to a canvas-style workspace.  
- Draft writing and interactive tool building move into search.  

- This shift overlaps with productivity tools, agents, and toolchains.  
- Execution, history, and verification may matter more for trust.  
- Sources and reproducibility become more visible criteria.  

- If you use AI Mode, you can try leaving deliverables on the canvas.  
- You can first locate settings for history, network, and execution.  
- Then you can pilot the workflow with small tasks.  

## Current status
An excerpt from Google’s blog post says Canvas in AI Mode is “available for everyone in the U.S. in English.”  
The excerpt also says Canvas can “draft documents” and “build interactive tools.”  
This section relies on the excerpt.  
More confirmation could help clarify the rollout details.  
It is unclear whether this is a Labs opt-in or a broader rollout.  
The Verge described an AI Mode “opt-in” condition.  

The Verge reported that US desktop users who have opted into AI Mode in Google Labs can use Canvas to create and refine travel plans in a side panel.  
The Verge also wrote that created plans are saved to AI Mode history.  
The word “saved” is important.  
It implies work can continue across sessions.  
It suggests history and version management may become central.  

For comparison, OpenAI materials describe ChatGPT Canvas features.  
They say Canvas can “render React/HTML code in a sandbox environment.”  
They also say code execution and rendering can make external network requests.  
They describe enterprise toggles for “allow code execution” and “allow network access.”  
They state a default of code ON and network OFF.  
There is no evidence that this model applies to Google Search Canvas.  
This comparison is only a reference point.  

## Analysis
Canvas looks like a change in the unit of work.  
Older search AI often followed question, then answer, then a new question.  
Users then copied results into other tools.  
Canvas can reduce that switching cost.  
Search can become an interface that occupies work time.  
Competitive criteria may shift with that change.  
Text plausibility may matter less than editability and persistence.  
Draft reliability across revisions becomes more important.  

Risks may also increase.  

First, verification.  
The research results provide limited evidence of reproducibility promises.  
They also provide limited evidence of standardized citation rules.  
Canvas can speed drafting, but drafts can still be wrong.  

Second, data and permissions.  
The Verge noted that plans are saved to AI Mode history.  
That raises questions about retention, reuse, and opt-out paths.  
This research does not establish policy changes in one place.  
It also does not provide a clear before-and-after comparison.  

Third, the execution environment.  
“Interactive tools” could imply code execution.  
If so, sandboxing and network access controls become relevant.  
OpenAI documents mention sandboxing and possible network requests.  
They also mention defaults of code ON and network OFF for enterprises.  
It remains unclear whether Search Canvas discloses similar controls.  
More confirmation would help.  

## Practical application
From a user perspective, the shift is toward a deliverables list.  
It is less like a question list.  
The canvas keeps drafts, not only a chat log.  
Prompts can shift toward edit instructions.  
One pattern is: set structure, then show evidence.  
Then convert blanks into questions.  
Then extract next steps as a checklist.  

If you are a developer or planner, treat interactive tools cautiously.  
You can start with prototypes rather than production apps.  
Assistive tools can include calculators and small simulators.  
Form-like utilities can also fit.  
If possible, confirm disclosures and restrictions in documentation.  
Key areas include execution, network access, and data access.  
This research does not pin down Google Search Canvas permissions.  

**Checklist for Today:**
- Find where AI Mode Canvas drafts are stored, including AI Mode history, and how deletion works.  
- Run one draft task end-to-end using outline, body, evidence check, and final summary steps.  
- Start interactive tools with local input values, and avoid network requests until controls are clear.

## Conclusion
The competitive focus for search may shift toward finishing work inside search.  
A workspace model increases the importance of storage and verification.

Before relying on Canvas, check history visibility and policy disclosures.

# Singularity Blog Writing Template

참고: MIT Technology Review, TechCrunch, Simon Willison's Blog

## 글 유형별 템플릿

### 1. 뉴스형 (News)
속보, 발표, 출시 소식

```
## 구조
1. 리드 (1-2문장): 무엇이, 언제, 누가
2. 핵심 (2-3문단): 왜 중요한지
3. 배경 (2-3문단): 맥락과 히스토리
4. 반응/전망 (1-2문단): 업계 반응, 의미
5. 출처 링크

## 톤
- 중립적, 사실 중심
- 짧은 문장 (15단어 이내)
- 능동태 사용
```

### 2. 분석형 (Analysis)
트렌드, 비교, 심층 분석

```
## 구조
1. 핵심 주장 (1문단): 이 글이 말하고자 하는 것
2. 현황 (2-3문단): 현재 상황, 데이터
3. 분석 (3-4문단): 왜 이런 일이 일어나는지
4. 의미 (2문단): 독자에게 어떤 영향이 있는지
5. 출처 링크

## 톤
- 분석적이지만 접근 가능
- 데이터와 예시 포함
- 개인 의견은 명확히 구분
```

### 3. 오피니언형 (Opinion)
관점, 제안, 비평

```
## 구조
1. 훅 (1-2문장): 독자의 관심을 끄는 시작
2. 주장 (1문단): 핵심 논점
3. 근거 (3-4문단): 주장을 뒷받침하는 증거
4. 반론 (1문단): 다른 관점 인정
5. 결론 (1문단): 행동 제안 또는 요약
6. 출처 링크

## 톤
- 확신 있지만 독선적이지 않게
- "~라고 생각한다" 대신 직접적 진술
- 근거 없는 주장 금지
```

---

## 스타일 가이드

### 헤드라인
- 8-12단어
- 능동태, 현재 시제
- 클릭베이트 금지
- 핵심 키워드 포함

```
Good: "OpenAI Releases GPT-5 With Native Tool Integration"
Bad: "You Won't Believe What OpenAI Just Did!"
```

### 첫 문단 (리드)
- 2-3문장
- 누가, 무엇을, 왜 중요한지
- 독자가 더 읽고 싶게

```
Good: "Google은 어제 Gemini 3를 발표했다. 6조 파라미터 모델로, 기존 최고 성능을 30% 앞섰다."
Bad: "AI 분야에서 많은 일이 일어나고 있다. 오늘은 그 중 하나를 소개하려고 한다."
```

### 문단
- 2-4문장
- 한 문단 = 한 아이디어
- 전환어로 연결

### 데이터 표현
- 구체적 숫자 사용
- 출처 명시
- 맥락 제공

```
Good: "LMArena 기준 1483 ELO로 1위를 차지했다. 2위와 31점 차이다."
Bad: "매우 높은 점수를 받았다."
```

### 인용과 출처
- 인라인 링크 선호
- 주장마다 근거 제시
- 글 끝에 출처 목록

---

## 피해야 할 것

1. **모호한 표현**: "매우", "정말", "엄청난"
2. **과장**: "혁명적", "게임체인저", "역사상 처음"
3. **근거 없는 단정**: "100%", "완벽", "절대", "반드시", "이론적으로 불가능"
4. **근거 없는 예측**: "~할 것이다" (출처 없이)
5. **수동태 남용**: "발표되었다" → "X가 발표했다"
6. **긴 문장**: 20단어 넘으면 쪼개기
7. **중복**: 같은 정보 반복 금지

---

## 체크리스트

글 작성 후 확인:

- [ ] 헤드라인이 내용을 정확히 반영하는가?
- [ ] 첫 문단에서 핵심을 알 수 있는가?
- [ ] 모든 주장에 근거가 있는가?
- [ ] 문단이 4문장 이하인가?
- [ ] 출처 링크가 작동하는가?
- [ ] 마크다운 서식이 일관적인가?

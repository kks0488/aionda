# AI온다 글쓰기 템플릿 (시리즈 3종)

참고: MIT Technology Review, TechCrunch, Simon Willison's Blog

이 문서는 **콘텐츠 “형식”을 표준화**해서 품질/SEO/유입을 안정적으로 올리기 위한 SSOT입니다.

## 시리즈 3종 (운영용)

아래 3개 중 하나를 선택해서 씁니다. (새 기능이 아니라 **같은 파이프라인 출력 품질을 표준화**하는 규칙)

### 1) K‑AI Pulse (Signal Brief)

- 목적: 빠르게 읽히는 “신호” 전달 + 공유/재방문 유도
- 적합: 발표/업데이트/논쟁/정책 변화 등 **시의성 높은 토픽**
- 톤: 짧고 단단하게, 과장 없이 “무슨 변화인가 / 왜 중요한가 / 뭘 하면 되나”

**권장 구성(한국어 기준)**  
> 기존 파이프라인 섹션(세 줄 요약/현황/분석/실전 적용/FAQ)을 유지하되, 각 섹션을 **짧게** 쓴다.

### 2) Explainer (Pillar / Evergreen)

- 목적: 검색 유입(에버그린) + 내부링크 허브(클러스터) 역할
- 적합: 개념/방법론/벤치마크 읽는 법/용어 정리/프레임워크
- 톤: “처음 읽는 사람” 기준으로 정의→예시→오해→실전 체크리스트

**권장 구성**
- 현황: “무엇을 설명하는 글인지” + 핵심 개념 정의(짧게)
- 분석: 오해/함정/트레이드오프를 최소 1개는 다룬다
- 실전 적용: 체크리스트는 “오늘 바로”가 아니라 “반복 가능한 습관”으로 쓴다

### 3) Deep Dive (Decision Memo)

- 목적: 깊이/신뢰/북마크 유도 + “왜”를 남기는 글
- 적합: 기술/정책/시장 변화의 영향, 비교, 전략, 리스크 분석
- 톤: 주장보다 근거, 단정보다 조건(If/Then), 균형(찬반/한계) 강조

**권장 구성**
- 현황: 사실(확인된 것) vs 미확인(추가 확인 필요)을 명확히 분리
- 분석: 2개의 상충하는 해석/시나리오를 제시하고, 각각의 조건을 쓴다
- 실전 적용: “실험 설계/검증 루프” 형태의 체크리스트(3개)로 끝낸다

## 스타일 가이드

### 헤드라인
- 8-12단어
- 능동태, 현재 시제
- 클릭베이트 금지
- 핵심 키워드 포함

```
Good: "OpenAI Releases GPT-5 With Native Tool Integration"
Bad: "You Won't Believe What OpenAI Just Did!"
```

### 첫 문단 (리드)
- 2-3문장
- 누가, 무엇을, 왜 중요한지
- 독자가 더 읽고 싶게

```
Good: "Google은 어제 Gemini 3를 발표했다. 6조 파라미터 모델로, 기존 최고 성능을 30% 앞섰다."
Bad: "AI 분야에서 많은 일이 일어나고 있다. 오늘은 그 중 하나를 소개하려고 한다."
```

### 문단
- 2-4문장
- 한 문단 = 한 아이디어
- 전환어로 연결

### 데이터 표현
- 구체적 숫자 사용
- 출처 명시
- 맥락 제공

```
Good: "LMArena 기준 1483 ELO로 1위를 차지했다. 2위와 31점 차이다."
Bad: "매우 높은 점수를 받았다."
```

### 인용과 출처
- 주장마다 근거 제시(가능하면 1차/주요 매체 우선)
- 출처 섹션은 시스템이 자동으로 붙이므로 본문에 별도 “참고 자료” 섹션을 만들지 않는다.

## 피해야 할 것

1. **모호한 표현**: "매우", "정말", "엄청난"
2. **과장**: "혁명적", "게임체인저", "역사상 처음"
3. **근거 없는 단정**: "100%", "완벽", "절대", "반드시", "이론적으로 불가능"
4. **근거 없는 예측**: "~할 것이다" (출처 없이)
5. **수동태 남용**: "발표되었다" → "X가 발표했다"
6. **긴 문장**: 20단어 넘으면 쪼개기
7. **중복**: 같은 정보 반복 금지

# AI온다 (aionda) - 비전과 철학

## Mission Statement

> **"한국 AI 커뮤니티의 집단지성을 전문적인 콘텐츠로 정제하여 세계에 전파한다"**

---

## 우리가 하는 것 vs 하지 않는 것

### ✅ 우리가 하는 것

| 단계 | 설명 |
|------|------|
| **큐레이션** | 수백 개의 갤러리 글 중 **정말 가치있는 정보**만 선별 |
| **검증** | AI + 웹 검색으로 **사실 여부 확인**, 거짓/과장 정보 필터링 |
| **재구성** | 커뮤니티 글을 **전문적인 블로그 아티클**로 변환 |
| **번역** | 한국어 정보를 **영어로 번역**해서 글로벌 발행 |
| **시각화** | AI 생성 이미지로 **프로페셔널한 외관** 제공 |

### ❌ 우리가 하지 않는 것

- 갤러리 글을 **그대로 복사**해서 올리는 것
- 품질 검증 없이 **양**만 채우는 것
- "제목 없음", 한 줄 채팅 같은 **쓰레기 글** 발행
- 사실 확인 없이 **루머/추측** 전파
- 단순 **번역기 돌린** 저품질 영어 콘텐츠

### 우리의 가치

1. **신뢰성**: 검증된 정보만 발행 (verificationScore 0.5+)
2. **전문성**: 구조화된 아티클 형식 (서론-본론-결론-FAQ)
3. **글로벌**: 영어 번역으로 전세계 접근성
4. **시의성**: 최신 AI 소식 빠르게 전달
5. **큐레이션**: 수백 개 중 진짜 가치있는 것만

## 타겟 독자

### Primary (영어권)

- **AI 연구자/개발자**: 한국 AI 생태계 인사이트
- **테크 저널리스트**: 아시아 AI 트렌드 소스
- **AI 스타트업**: 한국 시장 동향 파악
- **일반 AI 팬**: AGI/특이점 담론

### Secondary (한국어권)

- **갤러리 이용자**: 정제된 버전으로 다시 보기
- **AI 관심 일반인**: 접근성 높은 콘텐츠

## 콘텐츠 원칙

### 1. 품질 > 양

```
BAD:  하루 20개 쓰레기 글
GOOD: 하루 3-5개 고품질 글
```

**기준:**
- 최소 500자 이상의 본문
- 검증 가능한 주장 포함
- 의미있는 제목 (not "제목 없음")
- 2개 이상의 문단/문장

### 2. 검증된 정보만

```
BAD:  "GPT-6 내일 출시된대!" (루머 그대로)
GOOD: "OpenAI 공식 블로그에 따르면..." (출처 확인)
```

**검증 티어:**
- S: 공식 블로그 (openai.com, anthropic.com)
- A: 주요 언론 (Reuters, TechCrunch)
- B: 기술 블로그 (Medium, Dev.to)
- C: 기타 (낮은 신뢰도)

### 3. 전문적 재구성

```
BAD:  갤러리 글 그대로 복붙
GOOD: 구조화된 아티클로 변환
```

**구조:**
```markdown
# 제목 (명확하고 SEO 최적화)

[리드 문단: 핵심 정보 요약]

## 주요 내용
[상세 설명]

## 배경/맥락
[왜 중요한지]

## 향후 전망
[영향/의미]

## FAQ
[자주 묻는 질문]
```

### 4. 프로페셔널 외관

- AI 생성 커버 이미지 (모던, 테크 스타일)
- 일관된 브랜딩
- 깔끔한 레이아웃
- 다크모드 지원

## 품질 게이팅 시스템

### Phase 1: 수집 (Crawl)

```
DC Inside Gallery
      │
      ▼
  440+ 원본 글
```

### Phase 2: 1차 필터 (Auto-Select)

```
필수 조건 (즉시 거부):
  ✗ 제목이 "제목 없음", "무제" 등
  ✗ 본문 500자 미만
  ✗ 유의미한 문장 2개 미만
  ✗ 이모티콘/기호만 있는 글

점수 계산:
  + 조회수/50 (최대 20점)
  + 좋아요×2 (최대 20점)
  + AI 키워드: +5점/개
  + 정보/뉴스 카테고리: +10점
  - 스팸 키워드: -15점/개
  - 짧은 제목: -20점

통과 기준: 30점 이상
      │
      ▼
  ~3개/실행 선별
```

### Phase 3: 검증 (Verify)

```
검증 프로세스:
  1. 클레임 추출 (AI)
  2. 웹 검색으로 출처 확인
  3. 소스 티어 분류 (S/A/B/C)
  4. 검증 점수 계산

No-Claims 처리:
  - 검증할 주장 없음 = 저신뢰도 (0.3)
  - 의견/잡담으로 간주

통과 기준: verificationScore 0.5+
      │
      ▼
  검증된 글만
```

### Phase 4: 구조화 (Structure)

```
AI 구조화:
  1. 글 유형 분류 (뉴스/분석/의견)
  2. 전문적 구조로 재작성
  3. 헤드라인 생성

쓰레기 타이틀 금지:
  - AI가 "제목 없음" 반환 → 원본 제목 사용
  - 원본도 쓰레기 → 발행 거부

      │
      ▼
  전문 아티클
```

### Phase 5: 최종 검증 (Generate-Post)

```
이중 검증:
  1. 원본 제목 검증
  2. AI 생성 제목 검증
  3. 본문 길이 재확인
  4. 검증 점수 재확인

어느 하나라도 실패 → 발행 거부
      │
      ▼
  MDX 생성
```

## 왜 이 블로그가 가치있는가?

### 1. 정보 비대칭 해소

한국 AI 커뮤니티에는 영어권에서 접하기 어려운 인사이트가 있습니다:
- 한국 AI 스타트업 동향
- 아시아 시장 관점의 AI 분석
- 한국어 특화 LLM 소식
- 특이점주의자들의 담론

### 2. 집단지성의 정제

갤러리에는 쓰레기도 많지만, 가끔 **진짜 가치있는 정보**가 있습니다:
- 전문가 수준의 기술 분석
- 빠른 속보 (공식 발표 직후)
- 독특한 관점의 의견

우리는 이것들을 **찾아내고, 검증하고, 정제**합니다.

### 3. 접근성 향상

- 한국어 → 영어 번역
- 커뮤니티 형식 → 블로그 형식
- 분산된 정보 → 한 곳에 모음

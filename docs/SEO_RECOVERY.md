# AI온다 검색 품질 복구 계획

## 문제 진단

- 누적 글 2,390개의 대부분이 같은 저자와 반복된 목차를 사용했다.
- `세 줄 요약 → 현황 → 분석 → 실전 적용 → FAQ`가 주제와 무관하게 반복됐다.
- 운영 모델은 GPT-5.4였지만, 긴 절차형 프롬프트가 모델의 편집 판단을 제한했다.
- `gpt-5.6-sol`은 현재 로컬 프록시 모델 목록에는 있지만 인증 라우트가 없어 503을 반환했다.
- 실제 호출을 통과한 품질 우선 모델 `gpt-5.5`를 작성에, `gpt-5.6-terra`를 조사에 사용한다.

## 반영된 운영 정책

1. 글 작성은 `gpt-5.5` + `medium` reasoning을 기본으로 사용한다.
2. 프롬프트는 성공 기준과 근거 계약을 중심으로 짧게 유지한다.
3. TL;DR, FAQ, 체크리스트를 모든 글에 자동 삽입하지 않는다.
4. 조사 신뢰도 0.80 미만 주제는 작성하지 않는다.
5. 기존 글은 삭제하지 않고 가역적 색인 품질 정책을 적용한다.

## 현재 색인 분류

`pnpm seo:audit`로 재현할 수 있다. 기본은 `SEO_QUALITY_PHASE=1`로 검증 점수 0.75 미만, 엷은 본문, 낮은 점수와 출처 부족이 겹친 고위험 글만 제외한다. `SEO_QUALITY_PHASE=2`는 Search Console 데이터 검토 후에만 사용한다.

- 전체: 2,390
- 색인 유지/제외 수는 `pnpm seo:audit`의 현재 출력을 기준으로 한다.

`noindex` 페이지도 독자가 URL로 접근할 수 있다. 사이트맵에서만 제외하고 검색엔진에 `noindex, follow`를 제공한다.

## 다음 운영 단계

- Search Console에서 수동 조치, 크롤링됨-현재 색인되지 않음, 검색어별 노출을 기록한다.
- 노출이 있는 `noindex` 후보는 재편집 우선순위로 올린다.
- 한 주제의 유사 글은 핵심 가이드 하나로 통합하고 기존 URL은 리디렉션한다.
- 4주간 신규 글을 표본 평가한 뒤 reasoning 수준과 발행 속도를 재조정한다.
